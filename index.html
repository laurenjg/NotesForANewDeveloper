where to start
OOP Object oriented programming is a way of writing and organising
your code into objects that encapsulate data and interact
with one another. Benefits of programming with objects:
- Groups related data and actions
- Makes code 'objects' reusable
- Makes program easier to understand (in theory)
- Makes program easier to update, test, debug, fix and maintain

Every 'object' has:
- Properties (data)
- Methods (actions)

What is an 'object'?
Think of an 'object' as a 'unit' of code, in JS this could be a collection
of key-value pairs (an object or map), in C# this could be an interface
(more on interfaces later)

What does it mean when a program is 'object oriented'?
- The code is made up of objects interacting
Are there other ways of programming, if so what are they and how do they differ?
- Yes, for contextual understanding, there are multiple different programming
approaches for different including:
- Procedural programming
Code is essentially just a sequence of instructions

- Functional programming
Code focuses on pure mathematical functions
Think python

- Declarative programming
Code describes exactly what you want done and how to do it
Think SQL, HTML, CSS

- Event-driven programming
Code responds to events like user actions, a common web development practice
Commonly implemented using OOP concepts

- Component-based programming
Code is made up of reusable components
Thing, react or angular
Commonly implemented using OOP concepts
